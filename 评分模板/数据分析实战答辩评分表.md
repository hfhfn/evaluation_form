## 评分维度

### 维度1：项目完整程度（所有需求是否准确完成）
> 加分项，如果没有不计分，如果有加分

| 等级                           | 分值 |
| ------------------------------ | ---- |
| 多项关键需求未完成             | 1 分 |
| 完成核心需求，有少量需求未实现 | 2 分 |
| 项目完整实现所有明确需求       | 3 分 |

---

### 维度2：项目答辩表述清晰度

| 等级                                   | 分值 |
| -------------------------------------- | ---- |
| 讲解卡顿，逻辑混乱                     | 1 分 |
| 基本清晰，偶有停顿或逻辑跳跃           | 2 分 |
| 表述清晰，从需求到设计再到实现逻辑连贯 | 3 分 |

---

### 维度3：PPT设计是否清晰明了
> 避免大篇幅粘贴代码，要有代码和结果展示的说明

| 等级                                                         | 分值 |
| ------------------------------------------------------------ | ---- |
| 几乎全是代码或几乎无结果说明，逻辑混乱                       | 0 分 |
| 代码和结果展示不足，但思路能看懂                             | 1 分 |
| 代码和结果显示清楚，说明和细节基本完整                       | 2 分 |
| 关键代码片段 + 结果截图(FineBI) + 文字说明清晰对应：结构逻辑强，章节清晰完整 | 3 分 |

---

### 维度4：团队协作与分工

| 等级                                           | 分值 |
| ---------------------------------------------- | ---- |
| 分工不明确，职责不清晰没有体现                 | 1 分 |

| 团队中有分工协作或每名成员都有不同的工作量体现 | 3 分 |
