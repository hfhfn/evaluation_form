# 实战答辩评分系统

一个轻量的局域网**答辩互评**系统：学生在手机/电脑上为其他小组打分，管理员后台管理班级名单、评分标准与成绩汇总。基于 FastAPI，默认使用 SQLite，开箱即用。

---

## ✨ 功能特性

- **学生评分页**（现代全屏弹窗）
  - 输入姓名核对名单后锁定身份，自动识别所在组
  - 自动按**本班级的评分表**逐维度打分，实时显示总分与进度，可填写评语
  - 每组限评一次、不能给本组评分，已评可回看
- **管理后台**
  - 班级管理：新建班级（支持空班级，**建班即生成该班独立评分表**）、按班级筛选、删除班级（含数据清理）
  - 学生名单：粘贴 / 文件（JSON、CSV）/ 手动导入，按"姓名+班级"去重
  - 评分标准：**每个班级各有一套评分表，互不影响**；可选择"全局默认/某班级"分别编辑；可从模板加载后修改再保存
  - 评分模板：把当前标准存为模板，随时加载复用，或用模板直接建班
  - 成绩汇总：**按班级独立查看**（页面自带班级选择器）；总排名、各维度平均分（保留 2 位小数）、评分明细，一键**导出 Excel**
  - **重置评分**：成绩汇总→查看明细，可单独重置（删除）某个学生对某组的评分，该生随后可重新评分
  - **清空本班评分**：一键清空某班全部评分（保留学生名单与评分标准），用于更换评分标准后重新评分
  - 系统设置：修改管理员密码
- **每班独立评分表**：不同班级可用不同评分标准；改某班的表不影响其他班，也不影响历史成绩（维度名快照保存，成绩汇总/导出按**维度名称**匹配，换标准也不丢历史维度分）
- **提交防误/防篡改**：服务端按本班评分表校验每次提交（维度须全选、选项须属于本班、分数一律以数据库为准）；学生端某组全选最高档时会二次确认
- **双数据库**：默认 SQLite，零配置；也可切换 MySQL

---

## 🧱 技术栈

- 后端：Python 3.10+ / FastAPI / Uvicorn
- 存储：SQLite（默认）或 MySQL（PyMySQL）
- 前端：原生 HTML/CSS/JS（无构建步骤，直接由后端托管）

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

> 运行端到端测试需要 `httpx`；使用 MySQL 需要 `pymysql`（见下方）。

### 2. 启动（默认 SQLite）

```bash
python main.py
```

启动后控制台会打印访问地址，例如：

```
==================================================
  实战答辩评分系统已启动
==================================================
  学生评分:  http://192.168.x.x:8888
  管理员:    http://192.168.x.x:8888/admin/login
  数据库:    SQLite
==================================================
```

- 学生端：`http://<本机IP>:8888`
- 管理后台：`http://<本机IP>:8888/admin/login`

> 数据（成绩库 `evaluation.db` 与会话）默认存放在项目下的 `data/` 目录。

### 用 Docker 启动（推荐，免每次开终端）

```bash
docker compose up -d          # 构建并后台启动（开机自启）
docker compose up -d --build  # 改了代码后重建
docker compose down           # 停止
docker compose logs -f        # 看日志
```

- 访问地址同上：`http://<本机IP>:8888`。用 Docker 时控制台不会打印地址，请用**宿主机**局域网 IP（`ipconfig` 查 IPv4）。
- 数据挂载在 `./data/`，容器重建/升级都不丢；`python main.py` 与 Docker **共用这一份数据**（勿同时启动，会抢 8888 端口）。
- 首次上线建议在 `docker-compose.yml` 里设置 `EVAL_ADMIN_PASSWORD`、`EVAL_SECRET_KEY`。

### 3. 默认管理员账号

| 用户名 | 密码 |
|--------|----------|
| `admin` | `admin123` |

> 登录后请到「系统设置」修改密码。

## ⚙️ 命令行参数

| 参数 | 说明 | 默认值          |
|------|------|--------------|
| `--db` | 数据库类型：`sqlite` / `mysql` | `sqlite`     |
| `--port` | 服务端口 | `8888`       |
| `--host` | 绑定地址 | `0.0.0.0`    |
| `--mysql-host` | MySQL 主机 | `localhost`  |
| `--mysql-port` | MySQL 端口 | `3306`       |
| `--mysql-user` | MySQL 用户 | `root`       |
| `--mysql-password` | MySQL 密码 | （空）          |
| `--mysql-db` | MySQL 数据库名 | `evaluation` |

### 使用 MySQL

```bash
pip install pymysql
python main.py --db mysql --mysql-user root --mysql-password 你的密码 --mysql-db evaluation
```

> 数据库需提前创建（`CREATE DATABASE evaluation DEFAULT CHARSET utf8mb4;`），表结构由程序自动建立。
> 注：项目主力测试路径为 SQLite，MySQL 路径请先用 `python test_e2e.py` 思路自测。

## 📖 使用流程

### 管理员

1. 登录后台 → 「学生管理」
2. 点 **➕ 新建班级**，输入班级名（空班级即可，系统会自动为它生成一套评分表）
3. 选中该班级，粘贴/上传/手动**导入学生名单**（格式见下）
4. 到「评分标准」，在顶部 **"✏️ 正在编辑"** 选择该班级，调整其评分维度
   - 可在"📋 从模板加载"选择模板 → **加载到编辑器** → 修改 → **💾 保存评分标准**
   - 每个班级的评分表互相独立，改一个不影响其他班
5. 答辩进行中，学生在手机端评分（自动用本班评分表）
6. 到「成绩汇总」按班级查看排名/明细，或**导出 Excel**（见下）

> 💡 不同班级可用不同评分表；建班时也可在创建接口指定模板（`template_id`）直接套用。
> 即便后续修改某班评分表，历史成绩的明细维度名也会保留（提交时已快照），不会丢失。
> 如遇个别异常评分，可在「查看明细」里点该行 **重置** 单独删除，该生可重新评分。
> 若要给某班更换评分标准后重新评分，先在成绩汇总页点 **🗑 清空本班评分** 清场（保留名单与标准），再重新评分即可。

### 导出 Excel（.xlsx）

「成绩汇总 → 📥 导出 Excel」下载一个含两个工作表的 Excel：

1. **评分明细** —— 每位评分人对每组的逐维度打分、总分、评语（含姓名，作为底稿）。
2. **排名与评语** —— 所有组的排名、各维度均分、综合平均分（2 位小数）；下方按组**逐条列出收到的评语（匿名，不含评分人姓名）**。

> 文件带 UTF-8 BOM，Windows 版 Excel 双击即为正确中文；逗号/换行/引号等特殊字符均正确转义，不会串列。

### 学生

1. 打开学生端地址
2. 输入真实姓名（需在管理员导入的名单中）→ 确认身份
3. 点其他小组的「开始评分」，按**本班评分表**逐维度选择等级、可写评语 → 提交（不可修改）

### 名单导入格式

**JSON：**
```json
[{"name": "张三", "group": 1}, {"name": "李四", "group": 2}]
```

**CSV / 从 Excel 复制：**
```
姓名,组号
张三,1
李四,2
```

## 📊 默认评分标准

默认内置 6 个维度（取自 `答辩评分模板.md`），满分 18 分：

| 维度 | 分值区间 |
|------|----------|
| 需求分析 + 技术架构 + 可行性分析 | 1~3 |
| 团队分工与协作 | 1~3 |
| PPT质量 + 讲解表现 | 1~3 |
| 问题发现与解决 | 1~3 |
| 项目完成度 | 1~3 |
| 情感分（团队认同 / 态度 / 氛围） | 0~3 |

> 这是**全局默认评分表**，新建班级会以它为蓝本复制一份。维度与分值均可在「评分标准」页按"全局默认 / 各班级"分别修改。

## ❓ 常见问题

- **学生提示"名单中未找到"**：该姓名未被管理员导入，或导入到了别的班级。
- **新建的空班级看不到学生**：空班级正常显示（0 人），导入学生后人数即更新。
- **不同班级想用不同评分表**：到「评分标准」页顶部"✏️ 正在编辑"切换班级分别设置即可；各班互不影响。
- **改了某班评分表会影响历史成绩吗**：不会。提交评分时已快照维度名，历史明细始终可读；编辑也会尽量保留维度编号。
- **手机访问不了**：确认手机与服务器在同一局域网，且防火墙放行了对应端口（默认 8080）。
